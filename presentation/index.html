Step 01 · Claude.ai

Brief the AI — and ask it to write the prompt.

Here's the clever bit: we don't ask Claude to build the app yet. First we paste this simple brief — it asks Claude to write us the detailed master prompt. Like briefing an architect, who then draws the blueprint.

Briefing prompt — paste into Claude first
I want to make a personal expense tracker app called "Spendly" — with a dashboard, charts, budgets, savings goals, and an AI advisor. Don't build it yet. First, write me a detailed prompt that I can paste back to you to create the whole app as a single HTML file. Just give me the prompt to copy.
✨

Claude reads this brief and writes back a long, precise master prompt — that's the blueprint we use in Step 2.

📝 Plain English in 📐 Detailed prompt out 🎯 Be specific about features

Step 02 · Claude.ai

Paste the prompt back — get the whole app.

1
Open claude.ai and start a new chatSign in with Google — a free account is all you need.

3
Let Claude workIt writes the entire app — this can take a little while. A very long output is completely normal; it's building everything.
4
Copy the code, save as index.htmlPaste into Notepad, save as index.html, and open it in any browser to test.

If Claude pauses mid-way, just reply “continue” and it will finish. Very long outputs sometimes get cut off — totally normal.

The secret sauce

The Master Prompt.

This exact brief produced the whole app. Notice how specific it is — that's why it works.

Master Prompt — paste into Claude
Build a premium personal finance web app called "Spendly" as a single self-contained HTML file. Use Chart.js 4.4.1 from cdnjs and Outfit + DM Mono from Google Fonts. No other external dependencies. ━━━ 🎨 DESIGN SYSTEM ━━━ CSS variables only — never hardcode colors: --bg:#080d14 --bg2:#0e1420 --bg3:#151c2c --bg4:#1c2438 --card:#111827 --border:#1e2d45 --border2:#243350 --violet:#7c6bff --violet2:#6254e8 --violet-glow:rgba(124,107,255,0.18) --emerald:#0fda85 --rose:#ff4d6a --amber:#ffb800 --sky:#38bdf8 --text:#e8edf5 --text2:#8fa3be --text3:#4a637e Font: Outfit for all UI, DM Mono for amounts/codes. Background #080d14. Cards use #111827 with 1px solid #1e2d45 border, border-radius:20px, subtle drop shadow, and a lift + glow on hover. ━━━ 🏗️ LAYOUT ━━━ Fixed left sidebar (260px wide, #0e1420 bg) + scrollable main area. Sidebar contains: • Logo: gradient violet square icon + "Spendly" wordmark (violet accent on "ly") • Nav items with emoji icons, active state = violet gradient bg + violet border • Section labels in uppercase 10px tracking • Bottom user card with gradient avatar (violet→emerald), name, email Sticky topbar (70px, backdrop-filter:blur(20px)) with: • Hamburger (mobile) + page title left side • Month selector (‹ select ›) + "+ Add Transaction" button right side Mobile: sidebar slides in/out with dark overlay, hamburger shows at ≤800px. ━━━ 📄 PAGES (7 total, SPA via tab switching) ━━━ 1. Dashboard • 4 stat cards in a row: Balance (violet glow), Income (emerald), Expenses (rose), Savings Rate (amber). Each has an icon, animated counter, and a small sub-badge. • 2 more stat cards: Daily Avg Spend, Biggest Expense. • 2-column row: Donut chart (expense breakdown) + Spending by Category bar list with colored progress bars. • Recent transactions list (last 8). 2. Transactions • Filter row: search bar + All/Income/Expense pills + category dropdown + sort dropdown + 📥 Export CSV button. • Summary chips row (filtered totals). • Full scrollable transaction list, each item: category emoji icon (colored bg circle), title, category + note, edit/delete actions, amount (green +/red −), date + optional 🔄 recurring amber badge. 3. Analytics • Income vs Expense grouped bar chart (6 months). • Expense distribution donut chart + legend. • Cumulative cash flow line chart. • Top spending categories ranked list. • Daily spending heatmap (grid of day squares, color intensity = spend amount). 4. Budget • Budget overview bar chart + Budget Health Score gauge (canvas arc ring, 0–100, color: green≥80, amber≥60, red<60). • Category budget cards: progress bar turns amber at 75%, rose at 90%, label shows ₹spent/₹limit, over-budget chip. 5. Goals • Goal cards: emoji + name, % badge, animated progress bar (violet fill), ₹saved / ₹to-go / days left footer, inline top-up input + "Add" button, "🏆 Goal Complete!" state. • Goals bar chart (saved vs target per goal). 6. Insights • Financial Health Score ring + interpretation label. • Personalized insight cards (auto-generated from actual data: savings rate, budget breaches, biggest category). • Spending forecast line chart for rest of current month. • 6-month income vs expense trend line. 7. AI Advisor • Header bar: gradient icon, title, green pulsing online dot. • Chat window (max-height:450px, scrollable): bot bubbles (dark card, top-left radius flat) + user bubbles (violet gradient, top-right radius flat), avatars, typing indicator (3 animated dots). • Quick-prompt chips: 'Analyze my spending', 'Am I on track with my goals?', 'How can I save more?', 'Where am I overspending?', 'Forecast my month-end balance'. • Textarea input + send button (violet gradient, arrow icon). • Enter to send, Shift+Enter for newline. • Right panel: Financial Snapshot card (income, expenses, balance, savings rate, tx count, goals count) + Tips card. • On send, call POST https://api.anthropic.com/v1/messages with model claude-sonnet-4-20250514, inject user's real transactions/budgets/goals as system context. Show typing indicator while awaiting response. ━━━ 📋 TRANSACTION MODAL ━━━ Full-screen overlay modal with: • Expense / Income toggle pills (rose active vs emerald active). • Amount input (large, centered, ₹ prefix). • Description input. • Category grid (emoji buttons, selected = violet border + bg glow). Categories: Food, Transport, Shopping, Entertainment, Health, Bills, Education, Travel, Housing, Freelance, Salary, Investment, Other. • Date picker. • Note input. • "🔄 Mark as recurring" checkbox row (amber accent, styled container). • Full-width "Save Transaction" button. • Escape to close, click outside overlay to close. ━━━ 💾 DATA & LOGIC ━━━ • localStorage for all state (spendly_tx, spendly_budgets, spendly_goals). • Seed 25+ realistic transactions across Apr–May 2025 on first load. • uid() = Date.now() + Math.random() string for IDs. • INR(v) formatter using Intl.NumberFormat('en-IN') with ₹ symbol. • All charts: dark theme, #1e2d45 grid lines, #4a637e tick labels, tooltips with dark bg. • renderAll() re-renders current page on any data change. • Keyboard: N = open Add Transaction modal (when no modal open and not typing), Escape = close any modal. • Toast notifications (bottom-right, slide up, ✅/❌ prefix, auto-dismiss 3s). • animateCount() for all stat numbers (700ms ease-out cubic). • CSV export: builds comma-separated string with headers, triggers download via Blob URL. ━━━ ✨ POLISH ━━━ • fadeIn page transition (opacity 0→1, translateY 8→0, 0.3s). • popIn card animation (cubic-bezier spring) for new list items. • Hover: cards lift translateY(-3px) with enhanced shadow + violet glow border. • Custom scrollbar: 6px, rounded, #243350 thumb. • All charts destroy previous instance before re-render to avoid canvas reuse errors. • Nav items close sidebar on mobile after selection.
✅

Why it works: it names the exact colours, the API, the chart library, the currency format, the keyboard shortcuts. The more exact your prompt, the better the result. Vague in = generic out.

The Lovable Prompt.

The full upgrade brief — real logins, a working AI advisor, mobile polish, and deployment — while keeping Spendly's exact look.

Lovable prompt — paste inside Lovable
I have uploaded an expense tracker web app called "Spendly". Upgrade it while keeping the existing design system unchanged (dark theme, glassmorphism, violet / emerald / rose / amber accents, Outfit font). ━━━ 1 · AUTH & CLOUD STORAGE ━━━ • Supabase Auth: Google sign-in, email/password, signup, password reset. • Store all user data in Supabase. • Row-Level Security so each user sees only their own data. ━━━ 2 · A REAL AI FINANCIAL ADVISOR ━━━ • Replace the placeholder advisor with a fully working one (use any AI provider you think best). • Analyse transactions, budgets, goals, categories, monthly trends, habits. • Give budget tips, savings strategies, overspending alerts, summaries, goal suggestions, insights. • Build a context system so it knows the full picture; support natural-language chat. ━━━ 3 · PREMIUM AUTH SCREENS ━━━ • Modern Login, Signup & Forgot-Password screens matching Spendly. • Validation, loading states, animations, polished errors. ━━━ 4 · MOBILE OPTIMISATION ━━━ • Fully responsive, mobile-first — layouts, nav, charts, forms, tables. ━━━ 5 · PRODUCTION READINESS ━━━ • Refactor, improve performance, remove dead code, fix edge cases. ━━━ 6 · DEPLOYMENT ━━━ • Deploy and provide a live public URL.
💡

The line “keep the design unchanged” is what stops Lovable from altering the look — you stay in control of the result.
